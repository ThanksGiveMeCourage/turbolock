### 详细代码（共 ~30 行）

#### ① `timerTask.reset()` — `timing_wheel.go`

```go
// 在 timerTask 结构体后新增

// reset 清空所有字段，为放回 sync.Pool 做准备。
// callback 必须置 nil，防止闭包引用导致 GC 无法回收。
func (t *timerTask) reset() {
    t.key = ""
    t.callback = nil
    t.ticks = 0
    t.cancelled = false
    t.prev = nil
    t.next = nil
}
```

#### ② `addTaskDirect()` — `timing_wheel.go`

```go
// 在 addTask() 方法后新增

// addTaskDirect 将已构造好的 task 挂入时间轮。
// 与 addTask 的区别：不 new timerTask，由调用方从 taskPool 获取。
func (tw *timingWheel) addTaskDirect(task *timerTask) {
    tw.mu.Lock()
    tw.insertTask(task)
    tw.mu.Unlock()
}
```

#### ③ 全局 Pool 变量 — `turbolock_impl.go`

```go
// 在 var ErrLockFailed 下面新增

var (
    ErrLockFailed = errors.New("turbolock: failed to acquire lock")
)

// ─── sync.Pool 对象池（阶段四：零内存分配） ───

var randPool = sync.Pool{
    New: func() any { return make([]byte, 32) },
}

var taskPool = sync.Pool{
    New: func() any { return &timerTask{} },
}
```

#### ④ `getValue()` — `turbolock_impl.go`

```go
// 修改前
func (t *defaultTurboLocker) getValue() (string, error) {
    b := make([]byte, 32)
    if _, err := rand.Read(b); err != nil {
        return "", err
    }
    return base64.StdEncoding.EncodeToString(b), nil
}

// 修改后
func (t *defaultTurboLocker) getValue() (string, error) {
    b := randPool.Get().([]byte)
    if _, err := rand.Read(b); err != nil {
        randPool.Put(b) // ← 出错也要放回，防止泄漏
        return "", err
    }
    s := base64.StdEncoding.EncodeToString(b)
    randPool.Put(b)
    return s, nil
}
```

#### ⑤ `Lock()` 中 addTask 处 — `turbolock_impl.go`

```go
// 修改前
var renewTask *timerTask
if t.opts.AutoRenew {
    acquiredAt := time.Now()
    renewInterval := t.opts.Expiry / 3
    renewTask = t.wheel.addTask(key, renewInterval, func(ctx context.Context) error {
        // ...续期逻辑...
    })
}

// 修改后
var renewTask *timerTask
if t.opts.AutoRenew {
    acquiredAt := time.Now()
    renewInterval := t.opts.Expiry / 3

    renewTask = taskPool.Get().(*timerTask)
    renewTask.key = key
    renewTask.ticks = int(renewInterval / tickMs)
    renewTask.cancelled = false
    renewTask.callback = func(ctx context.Context) error {
        if t.opts.MaxHoldDuration > 0 && time.Since(acquiredAt) > t.opts.MaxHoldDuration {
            renewTask.cancelled = true
            return nil
        }
        return t.client.Eval(ctx, renewLuaScript,
            []string{key}, value, int(t.opts.Expiry.Seconds())).Err()
    }

    t.wheel.addTaskDirect(renewTask)
}
```

#### ⑥ `fireSlot` cancelled 分支 — `timing_wheel.go`

```go
// 修改前
if task.cancelled {
    task = next
    continue
}

// 修改后

    task.reset()
```

### 分类判定

| 逃逸来源 | 含义 | 是否可接受 |
|---------|------|:--:|
| `context.Context` | 接口值天生逃逸 | ✅ 可接受 |
| `func literal escapes` | 闭包逃逸（续期回调） | ✅ 可接受（无法避免） |
| `github.com/redis/` | go-redis 内部 | ✅ 不在优化范围内 |
| `b := make([]byte, 32)` | rand buffer 还在逃逸 | ❌ 需检查 Pool 是否正确 |
| `&timerTask{}` | task 还在逃逸 | ❌ 需检查是否漏改 |

## 七、第六步：竞态检查

`sync.Pool` 的 `Get`/`Put` 配对错误会导致内存泄漏或数据竞争。

```bash
go test -race -run TestTimingWheel_Concurrent -count=1
go test -race -run TestAutoRenew_NoGoroutineLeak -count=1
```

**关注点**：

- `DATA RACE` 告警 → 立即排查 Get/Put 配对
- `WARNING` 无影响
- 零输出 = 通过

## 十、常见问题排查

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| allocs 没降 | 漏改了一处 `new(timerTask)` 或 `make([]byte,32)` | `grep -rn "new(timerTask)" *.go` |
| allocs 上升 | Pool 的 New 函数和 Get/Put 不配对 | 检查 `fireSlot` cancelled 路径 |
| DATA RACE | 两个 goroutine 同时 Put 同一个 task | 确认 `fireSlot` 持锁时 Put |
| B/op 不降 | 闭包捕获的变量（`value`, `key`）逃逸到堆 | 逃逸分析确认 |
